Core Concept: The Stateless Reactor

A simple reflex agent selects an action based **only on the current percept**, without any memory of the past. It operates on a direct condition-action rule, like a knee-jerk reaction. Its logic is stateless; the decision is based purely on the "now."

Example: The Automated Thermostat

Its only job is to turn a heater on or off based on the current room temperature. If it's cold, turn on. If it's warm, turn off. It doesn't remember if it was cold five minutes ago.

Mathematical Breakdown

The agent's behavior is a direct function \(f\) that maps the current percept (\(p_t\)) to an action (\(a_t\)).

\(a_t = f(p_t)\)
  • Percepts (P): The current temperature, \(p_t \in \mathbb{R}\).
  • Actions (A): \(\{ \text{HEATER\_ON}, \text{HEATER\_OFF} \}\).

Python Implementation

import time
import random

class SimpleThermostatAgent:
    """A simple reflex agent that acts based only on the current temperature."""
    def __init__(self, desired_temp):
        self.desired_temp = desired_temp

    def choose_action(self, current_percept):
        """Implements the function a_t = f(p_t)."""
        current_temperature = current_percept['temperature']
        if current_temperature < self.desired_temp:
            return 'HEATER_ON'
        else:
            return 'HEATER_OFF'

class RoomEnvironment:
    """Simulates a room where temperature changes."""
    def __init__(self, initial_temp):
        self.temperature = initial_temp

    def get_percept(self):
        return {'temperature': self.temperature}

    def update(self, action):
        if action == 'HEATER_ON':
            self.temperature += 0.5
        else:
            self.temperature -= 0.2
        self.temperature += random.uniform(-0.1, 0.1)

# --- Simulation ---
room = RoomEnvironment(initial_temp=18.5)
thermostat = SimpleThermostatAgent(desired_temp=20.0)

for step in range(5):
    percept = room.get_percept()
    action = thermostat.choose_action(percept)
    print(f"Temp: {percept['temperature']:.1f}¬∞C -> Action: {action}")
    room.update(action)
    time.sleep(1)

Core Concept: The Stateful Reactor

A model-based agent maintains an **internal state** (or "model") of the world to make decisions. This allows it to handle situations where the current percept alone isn't enough to act optimally. It "remembers" information.

Example: A Smarter Robot Vacuum

It senses dirt, but it also **remembers its own location**. If it senses a clean floor, its decision to move to another room depends on its internal memory of where it currently is.

Mathematical Breakdown

The agent uses a **state update function** (\(g\)) to maintain its internal model (\(s_t\)) and an **agent function** (\(f\)) that uses this state to choose an action.

\(s_t = g(s_{t-1}, a_{t-1})\)
\(a_t = f(s_t, p_t)\)
  • Internal State (\(S_{internal}\)): The agent's belief about its location, \(\{ \text{IN\_ROOM\_A}, \text{IN\_ROOM\_B} \}\).

‚Üí

The 'brain' now uses both memory AND percepts to decide.
‚Üí
Action (\(a_t\))
A more intelligent action based on context (e.g., 'Move to other room').

Python Implementation

import time
import random

class ModelBasedVacuumAgent:
    """A model-based agent that remembers its location."""
    def __init__(self, initial_location):
        self.internal_state_location = initial_location

    def update_state(self, last_action):
        """Implements s_t = g(s_{t-1}, a_{t-1})."""
        if last_action == 'MOVE_TO_B':
            self.internal_state_location = 'B'
        elif last_action == 'MOVE_TO_A':
            self.internal_state_location = 'A'

    def choose_action(self, percept):
        """Implements a_t = f(s_t, p_t)."""
        location_status = percept['status']
        if location_status == 'DIRTY':
            return 'SUCK'
        elif self.internal_state_location == 'A':
            return 'MOVE_TO_B'
        elif self.internal_state_location == 'B':
            return 'MOVE_TO_A'

class TwoRoomEnvironment:
    """Simulates a two-room world."""
    def __init__(self):
        self.room_status = {'A': 'DIRTY', 'B': 'DIRTY'}
        self.agent_location = 'A'

    def get_percept(self):
        return {'status': self.room_status[self.agent_location]}

    def update(self, action):
        if action == 'SUCK':
            self.room_status[self.agent_location] = 'CLEAN'
        elif action == 'MOVE_TO_B':
            self.agent_location = 'B'
        elif action == 'MOVE_TO_A':
            self.agent_location = 'A'

# --- Simulation ---
house = TwoRoomEnvironment()
vacuum = ModelBasedVacuumAgent(initial_location='A')
last_action = None

for step in range(5):
    if last_action:
        vacuum.update_state(last_action)
    percept = house.get_percept()
    action = vacuum.choose_action(percept)
    print(f"Location: {vacuum.internal_state_location}, Percept: {percept['status']} -> Action: {action}")
    house.update(action)
    last_action = action
    time.sleep(1)

Core Concept: The Future Planner

A goal-based agent goes beyond simple reactions by considering the **consequences of its actions**. It has a specific **goal**‚Äîa desirable state‚Äîand chooses actions that will help it achieve that goal. This often involves search and planning to find a sequence of actions.

Example: GPS Navigation

A GPS system knows your destination (the goal). It doesn't just make the next turn based on the current street; it calculates a full route (a sequence of actions) by simulating different paths to see which one best reaches the goal.

Logical Breakdown

The agent needs to know its **goal** and a **transition model** (how the world works). It then uses search algorithms to find a sequence of actions that leads to the goal state.

The core question is: "What sequence of actions will get me from my current state to my goal state?"

  • Goal (\(S_g\)): A set of desirable world states.
  • Transition Model (\(T\)): Predicts the next state based on the current state and an action. \(s' = T(s, a)\).
  • Planning: The process of finding a sequence of actions \([a_1, a_2, ..., a_n]\) to reach the goal.

Python Implementation

import time

class GoalBasedNavigationAgent:
    """A simple goal-based agent for navigating a 1D track."""
    def __init__(self, initial_position, goal_position):
        self.position = initial_position
        self.goal = goal_position
        print(f"Agent created at position {self.position}, goal is {self.goal}")

    def choose_action(self):
        """Plans a simple path to the goal."""
        # This is a very simple form of planning.
        if self.position < self.goal:
            return 'MOVE_RIGHT'
        elif self.position > self.goal:
            return 'MOVE_LEFT'
        else:
            return 'STAY'

class TrackEnvironment:
    """Simulates a 1D track."""
    def __init__(self, agent):
        self.agent = agent

    def update(self, action):
        if action == 'MOVE_RIGHT':
            self.agent.position += 1
        elif action == 'MOVE_LEFT':
            self.agent.position -= 1

# --- Simulation ---
agent = GoalBasedNavigationAgent(initial_position=3, goal_position=10)
track = TrackEnvironment(agent)

for step in range(10):
    if agent.position == agent.goal:
        print(f"Position: {agent.position} -> Goal Reached!")
        break
    
    action = agent.choose_action()
    print(f"Position: {agent.position} -> Action: {action}")
    track.update(action)
    time.sleep(0.5)

Core Concept: The Rational Decision-Maker

A utility-based agent acts not just on goals, but on **preferences**. When there are conflicting goals or when some goals are more important than others, it uses a **utility function** to quantify "happiness" or "desirability" for different world states. It then chooses the action that leads to the state with the highest **expected utility**.

Example: Ride-Sharing App Assistant

A personal assistant booking a ride has multiple factors to consider: cost, speed, and comfort. A fast, luxurious ride is expensive, while a slow, shared ride is cheap. A utility-based agent weighs these factors based on your preferences to choose the "best" option for you, not just the one that reaches the destination.

Mathematical Breakdown

The agent's decision is based on maximizing expected utility. It uses a utility function \(U(s)\) that assigns a real number to each state \(s\), representing its desirability.

The agent chooses the action \(a\) that maximizes the expected utility of the outcome states \(s'\).

\( \text{action} = \arg\max_a \sum_{s'} P(s' | s, a) \cdot U(s') \)
  • Utility Function (\(U(s)\)): A score for how "good" a state is.
  • Probability (\(P(s' | s, a)\)): The probability of reaching state \(s'\) from state \(s\) after taking action \(a\).

Python Implementation

import time

class UtilityBasedRoutingAgent:
    """An agent that chooses a travel route to maximize utility."""
    def __init__(self, time_preference, cost_preference):
        # Negative weights because time and cost are undesirable
        self.w_time = -abs(time_preference)
        self.w_cost = -abs(cost_preference)
        print(f"Agent created with preferences: Time={self.w_time}, Cost={self.w_cost}")

    def utility(self, time, cost):
        """Calculates the utility of a route."""
        return (self.w_time * time) + (self.w_cost * cost)

    def choose_action(self, options):
        """Chooses the route with the highest utility."""
        best_option = None
        max_utility = -float('inf')

        for name, stats in options.items():
            u = self.utility(stats['time'], stats['cost'])
            print(f"  - Evaluating '{name}': Utility = {u:.2f}")
            if u > max_utility:
                max_utility = u
                best_option = name
        
        return best_option

# --- Simulation ---
# Let's define some travel options
route_options = {
    'Fast_Toll_Road': {'time': 20, 'cost': 15},
    'City_Streets': {'time': 45, 'cost': 5},
    'Scenic_Route': {'time': 60, 'cost': 2}
}

# 1. An agent who values time highly
agent_in_a_hurry = UtilityBasedRoutingAgent(time_preference=10, cost_preference=2)
choice1 = agent_in_a_hurry.choose_action(route_options)
print(f"-> Agent in a hurry chose: {choice1}\n")

# 2. An agent who is trying to save money
frugal_agent = UtilityBasedRoutingAgent(time_preference=2, cost_preference=10)
choice2 = frugal_agent.choose_action(route_options)
print(f"-> Frugal agent chose: {choice2}")

Core Concept: The Improver

A learning agent is not designed with a fixed logic; it can **improve its performance over time** by learning from its experiences. It consists of a performance element (what it does) and a learning element (how it improves). This allows it to adapt to new or changing environments.

Example: Spam Filter

A spam filter starts with some basic rules. When you mark an email as "spam," the agent's **learning element** analyzes that email's features (words, sender, etc.) and updates the **performance element** to better detect similar spam in the future.

Logical Breakdown

A learning agent is structured into four main components that work together in a cycle of action and improvement.

  • Performance Element: The "actor." It uses its current knowledge to choose and perform actions. This is like the agent's 'body'.
  • Critic: The "evaluator." It observes the agent's performance and provides feedback (e.g., "that action led to a good outcome") to the learning element.
  • Learning Element: The "brain." It uses the critic's feedback to update the performance element's knowledge and rules, making it better over time.
  • Problem Generator: The "explorer." It suggests new or exploratory actions to the performance element to see what happens, leading to new learning experiences.

Python Implementation

import random

class LearningGuesserAgent:
    """An agent that learns to guess a number by narrowing its range."""
    def __init__(self):
        # Performance Element: The knowledge used to act.
        self.min_range = 0
        self.max_range = 100
        print("Agent created. Ready to learn!")

    def choose_action(self):
        """The Performance Element in action."""
        guess = random.randint(self.min_range, self.max_range)
        return guess

    def learn(self, last_guess, feedback):
        """The Learning Element: Updates knowledge based on feedback."""
        # The 'Critic' is the external feedback ('too_high' or 'too_low').
        print(f"  - Learning from feedback: Guess {last_guess} was {feedback}")
        if feedback == 'too_high':
            self.max_range = last_guess - 1
        elif feedback == 'too_low':
            self.min_range = last_guess + 1

# --- Simulation ---
agent = LearningGuesserAgent()
secret_number = 42

for i in range(10):
    # Agent's performance element makes a guess
    guess = agent.choose_action()
    
    # Environment provides feedback (acting as the Critic)
    if guess == secret_number:
        print(f"Guess {guess}: Correct! Agent learned in {i+1} tries.")
        break
    elif guess > secret_number:
        feedback = 'too_high'
    else:
        feedback = 'too_low'
        
    # Agent's learning element updates its knowledge
    agent.learn(guess, feedback)
